3. What is a raw type? Why is a raw type unsafe? Why is the raw type allowed in Java?

-> A raw type is a generic class or interface used without its type arguments.
-> It bypasses compile-time type checking.
   Because raw types treat all elements as Object, the compiler won't stop you from adding the wrong data type (e.g., adding a String to a list of Integers).
   This leads to a ClassCastException at runtime when you try to retrieve and cast the data.
-> It is allowed for backward compatibility.
   Generics were introduced in Java JDK 1.5. Raw types allow legacy code (written before 2004) to run and interact with modern, generic-based code without breaking.

4. What is erasure? Why are Java generics implements using erasure?
-> Erasure is a process where the Java compiler removes all generic type information during compilation.
-> Java uses erasure primarily for Backward Compatibility.
   Legacy Support: It allows modern code (JDK 1.5+) to work seamlessly with "legacy" libraries written for older versions (JDK 1.4 and earlier).
